# 贡献指南

首先感谢你对本项目的关注！欢迎以各种方式参与贡献。

## 如何贡献

### 报告 Bug

如果你发现了游戏中的 bug，请按以下方式提交：

1. 检查是否已有相关 Issue
2. 创建新 Issue 并详细描述：
   - 问题描述
   - 复现步骤
   - 预期行为
   - 实际行为
   - 浏览器和系统环境
   - 截图（如有）

### 功能建议

欢迎提出新功能建议：

1. 检查是否已有相关建议
2. 创建 Issue 说明你的想法
   - 这个功能解决了什么问题
   - 你期望的实现方式
   - 如果有参考游戏或截图也请附上

### 代码贡献

如果你想直接贡献代码：

1. Fork 本仓库
2. 创建你的功能分支 (git checkout -b feature/AmazingFeature)
3. 提交你的改动 (git commit -m 'Add some AmazingFeature')
4. 推送到分支 (git push origin feature/AmazingFeature)
5. 开启一个 Pull Request

## 代码规范

### JavaScript

- 使用 ES6+ 语法
- 变量和函数使用小驼峰命名
- 常量使用全大写下划线分隔
- 函数名应该清晰表达其用途
- 适当添加注释说明复杂逻辑

### HTML/CSS

- 使用语义化标签
- 类名使用 kebab-case
- 保持缩进一致（2空格）

## 游戏设计原则

在提交新功能时，请考虑以下设计原则：

1. **文化真实性**：尽量符合华夏历史文化背景
2. **游戏平衡性**：新功能不应破坏现有平衡
3. **可扩展性**：代码结构要便于后续扩展
4. **性能考虑**：保证游戏流畅运行
5. **纯前端**：不引入后端依赖，保持单文件可运行

## 开发环境

本项目无需特殊开发环境：

- 编辑器：任意文本编辑器（推荐 VS Code）
- 运行：直接用浏览器打开 index.html
- 调试：浏览器开发者工具

### 本地开发建议

```bash
# 使用 Python 启动本地服务器
python -m http.server 8080

# 或者使用 Node.js
npx serve .
```
